# 顧客差分ログ

> 目的：顧客ごとの「標準仕様と何が違うか」を記録する。
> これが将来の設定化・業種展開の判断材料になる。
> 顧客との会話・要望・不満を都度ここに追記していく。

---

## 記録のフォーマット

```
### [顧客名 or 仮ID]（業種・規模・開始日）

**要望・差分の内容**
- （例）「荷主」という言葉が自社では「得意先」と呼ばれている
- （例）運行実績に「積載量」の列が欲しい
- （例）経費申請の承認が2段階必要

**対応状況**
- 未対応 / custom_settings で吸収済み / 機能として追加済み

**設定化の可能性**
- 簡単（custom_settings のラベル変更で対応可） / 中程度 / 困難（DB設計変更が必要）
```
